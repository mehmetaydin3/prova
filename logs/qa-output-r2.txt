## FAIL Items — Exact Fix Instructions

### Check 1 & 2b — Token overrides / hardcoded colors in `MusicianListingPage.module.css`

**Root cause:** The `.page` block and nearly every rule below it bypasses the design token system with inline hex/rgba values. This also breaks dark mode since the page forcibly re-declares tokens rather than inheriting from `[data-theme="dark"]`.

**Fix:** Remove the entire `.page { ... }` variable override block (lines 6–27). The dark-luxe palette should be expressed via a scoped `[data-theme="dark"]` block in `tokens.css`. Every subsequent rule that uses a raw color literal must be replaced with the correct token reference, e.g.:
- `#111114` → `var(--color-surface-default)` (dark mode value)
- `#0D0D0F` → `var(--color-luxe-bg)` (already in tokens.css)
- `#F5F0E8` → `var(--color-text-primary)`
- `rgba(245,240,232,0.4)` → `var(--color-text-secondary)`
- `#F5A623` → `var(--color-luxe-amber)`
- `rgba(0,0,0,0.6)` overlay → `var(--color-surface-default)` at reduced opacity or a dedicated overlay token
