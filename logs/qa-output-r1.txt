## FAIL Items — Fix Instructions

### FAIL 1 & 2b — Token compliance + .page token overrides
**File:** `MusicianListingPage.module.css`

The `.page` block (lines 6–27) redefines 16 tokens with raw hex/rgba values instead of using the design system. Additionally, hardcoded colors appear throughout the file in selectors such as `.pageHeader` (`#0D0D0F`, `rgba(245,240,232,0.06)`), `.pageTitle` (`#F5F0E8`), `.filterCount` (`#F5A623`, `#0D0D0F`), `.categoryItemActive` (`#F5A623`), `.drawer` (`#111114`), etc.

**Fix:** Remove the entire `.page` token-override block (lines 6–27). Replace every hardcoded color value in the file with the appropriate token from `tokens.css` — e.g., `#F5F0E8` → `var(--color-text-primary)`, `#0D0D0F` → `var(--color-luxe-bg)`, `#F5A623` → `var(--color-luxe-amber)`, `rgba(245,240,232,0.06)` → `var(--color-surface-glass)`.
